## 1. Mission & Non-Negotiables

**Mission.** Apply the re-themed Crest web + mobile design (HTML/CSS/JS prototypes under `docs/`) onto the live Next.js 15 App Router app, producing pixel-faithful screens, a mobile app-shell, and an installable PWA — with **zero functional regressions**.

**Non-negotiable constraints (a violation of any one fails the release gate):**

1. **Platform-safety.** No auto-post / auto-DM / scraping anywhere. Reply drafts are **copy-only and disclosed**; the human posts them. Specifically preserve: the signup **disclosure consent checkbox**; the Settings **"Auto-post replies — Disabled by design"** affordance; the Sources **"Public sources / Official APIs only"** notice; copy/draft toast voice (*"Copied — now post it yourself"*, *"Draft generated — review before posting"*); the `SafetyNotice` / "No auto-posting" copy on dashboard, lead detail, billing.
2. **Demo-mode-zero-secrets.** App runs end-to-end with **no secrets**. The `auth-form.tsx` demo branch and the layout demo-mode banner stay intact. The PWA offline path precaches the demo shell only.
3. **Dual-store parity.** Only presentational layers change. **No** `getStore()` / server-action / data-flow edits. In-memory and Supabase stores stay behavior-identical.
4. **Server-side ownership + RLS.** No data-fetch or authorization changes. `requireContext()` guards stay. The service worker is **network-first for all `/api/*`** and never caches authenticated/RLS-scoped responses.
5. **Accessibility — WCAG 2.1 AA.** Reduced-motion guards on every ported animation; visible focus ring (token) preserved; 44px minimum touch targets on mobile; semantic HTML + ARIA; AA contrast on every text/background pair (spot-check Verdigris-on-ink dark wells).
6. **No functional regressions.** Server components stay server; add `"use client"` only where interaction demands it. **The e2e accessible names are contract** (see §4.F and §6).

**Presentation-only rule:** every change in this document is CSS/markup/className/layout. If a task appears to require touching data, auth, scoring, or store code — **stop and escalate**; it is out of scope.

## 3. Design-TOKEN Contract (single source of truth)

**Law:** Tokens are defined **once** as CSS custom properties in `src/app/globals.css :root`, exposed to utilities **only** via `var()` in `tailwind.config.ts`. **No magic values. No hard-coded colors** (`#hex`, `rgb()`, named Tailwind palette like `red-50`/`amber-50`/`slate-*`/`emerald-*`) in any `src/**` `.tsx`/`.css` except: (a) inline `<email>` CSS that cannot read vars, and (b) `<canvas>`/inline-SVG JS color objects that cannot read CSS vars at paint time (must carry a `// token: --x` comment). Geometry is **sharp 0px or full pill** only (`xs`=2px for inputs/chips). Depth is **1px border, never shadow** (`shadow-card: none`).

### 3.3 GRAIN — resolve the one real value conflict
Live `--grain` uses `opacity 0.5`; the DS spec (`docs/tokens/effects.css`) is `0.28`. **Canonical = `0.28`** (matches the new screenshots; subtler print tactility). Change the opacity in the `--grain` SVG data-URI in `globals.css` from `0.5` → `0.28`. The `crest-grain::before` overlay (currently `0.6`) and landing overlays scale proportionally — set the page grain to `0.28` and spot-check landing dark wells. Same noise SVG / `baseFrequency` otherwise.

### 3.4 Tailwind utilities to ADD in `tailwind.config.ts` (`theme.extend.colors`)
```
high-tint: var(--high-tint), medium-tint: var(--medium-tint), low-tint: var(--low-tint),
danger-tint: var(--danger-tint), info-tint: var(--info-tint)
```
So `bg-medium-tint`, `bg-info-tint`, etc. resolve for ported React components and replace the hard-coded bracket-hex in `src/lib/scoring/score.ts` and `src/app/app/leads/[id]/page.tsx`.

## 5. Engineering Rigor (enterprise expectations)

- **Rollout:** one feature branch off `main` (e.g. `crest-redesign`); land phases as stacked PRs (A→B→C→D→E→F). Gate the user-visible cutover behind a flag (`NEXT_PUBLIC_CREST_V2`) where a phase changes a whole screen's structure (inbox rows, auth split, app shell, mobile shell) so it can ship dark and flip atomically. Tokens (Phase A) and primitives (Phase B) are backward-compatible and need no flag.
- **Visual-diff CI gate:** Playwright screenshot snapshots for landing, dashboard, inbox, lead detail, auth, pricing, billing, settings, digest at desktop + mobile widths; diffs require human sign-off. Compare against `docs/screenshots/{m-dash,m-onb,m-views,onb-v2,signup-final,landing-final}.png` and `docs/uploads/pasted-*.png`.
- **Performance budgets:** LCP ≤ 2.5s (mobile, slow-4G), CLS < 0.1, route JS budget enforced. Keep server components server; add `"use client"` only for: `app-nav`, `auth-form`, `demo-search`, `notification-settings`, `extension-tokens`, `Toast`, mobile shell pieces, SW registrar, animation hooks. Lazy-load `ScanSheet`/radar; `lucide-react` tree-shaken. Self-host fonts via `next/font` (no runtime font fetch).
- **Reduced-motion:** every count-up/bar-fill/stagger/radar/pulse/spotlight/marquee honors `prefers-reduced-motion` (final state instant).
- **prefers-color-scheme:** Crest is light-first warm paper; honor the scheme for browser UI only (`theme-color`). **Dark "well" usage rule:** `.cell.ink`, `.terminal`, `.am-stat.ink`, `.plan.hi`, `.announce` are **intentional editorial inversions** (`var(--ink)` ground, `var(--paper)` text) — do **not** "fix" them; only verify AA on Verdigris-tint-on-near-black.
- **RTL-readiness:** use logical properties (`margin-inline`, `padding-block`, `inset-inline`) and `start/end` rather than `left/right` in new CSS where practical.
- **Semantic HTML + ARIA:** `<nav>/<main>/<header>`; tabs = `role="tablist"`/`tab`/`tabpanel`; toggles = real `<button aria-pressed>` / labeled `<input type=checkbox>`; toasts = `role="status"` `aria-live="polite"`; FAB has an accessible name; mono labels keep text content for SR.
- **ROLLBACK:** each phase is an isolated revertable PR. Tokens (A) revert by restoring `globals.css`/`tailwind.config.ts`. Structural screens (C/D) gated by `NEXT_PUBLIC_CREST_V2` — flip the flag off to fall back instantly with no redeploy. SW has a kill-switch (versioned cache name + `skipWaiting`/`clients.claim` and an unregister path) to purge a bad service worker.
