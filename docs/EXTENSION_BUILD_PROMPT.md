## 0. MANDATORY CORRECTIONS — apply before building

> Generated by a multi-agent design → synthesis → adversarial-safety-review
> workflow and reviewed (`needs-fixes`). Sections 1–10 are the full spec; **these
> overrides take precedence wherever they conflict.**

### Safety (boundary breaches — must fix)
1. **No wildcard host permissions.** The manifest must NOT request
   `optional_host_permissions: ["https://*/*"]` — a wildcard is functionally
   `<all_urls>` for `scripting` and breaks the Safety Contract. It has been
   removed from the Section 4 stub (left as `[]`). To support a self-hosted app
   origin, request the **single exact https origin** the user types **at runtime**
   via `chrome.permissions.request({ origins: [exactOrigin] })`, validated as
   https + single host — never a wildcard.
2. **No host-page injection (drop "Surface A" for v0.1).** Section 6's in-page
   button injects the extension's node into the host DOM, contradicting the
   "all UI renders in the extension's own popup/side-panel, never injected into
   the host page" invariant (B14). Ship **only the popup (Surface C)** for v0.1 —
   it already hosts the whole flow. Any future in-page affordance must be a single
   **closed-Shadow-DOM**, read-only, extension-owned node removed on close, with a
   narrow, explicit CI carve-out.

### Correctness (will bite the implementer — grounded in the repo)
3. **Migration is `0005_extension_tokens.sql`** — the repo is at
   `0004_notifications`, so the next number is 0005, not 0006.
4. **Capture is NOT idempotent as written.** `scoreManualPost` builds
   `external_id = manual_${…}_${Date.now()}`, so re-capturing the same post makes
   a **duplicate lead** and burns a `posts_scanned` credit each time. For the
   extension path, derive `external_id` from a **stable hash of `postUrl`** (drop
   `Date.now()`), or explicitly document that re-capture double-counts usage.
5. **`platform_reminder` is a server-synthesized constant**, not an AI field —
   `replyDraftSchema` has no such field. The draft route adds it; don't present it
   as flowing from the pipeline.
6. **Signed-in capture PERSISTS a lead** (raw_post + lead_candidate +
   usage_event). Scope every "nothing saved / transient" claim to the **demo /
   extension-local** layer only, and disclose — in-UI and in the privacy policy —
   that signed-in capture **saves a lead** that inherits `COMPLIANCE.md`
   cascade-deletion / removal handling.

### Hardening (add these)
7. **Token UPDATE RLS needs `with check (public.is_org_member(organization_id))`**
   (a `using`-only policy lets a member move a token to another of their orgs).
   Revocation is update-only (`revoked_at`); state hard deletes are denied or add
   a delete policy.
8. **Middleware MUST exclude `/api/extension`** — `src/middleware.ts` runs
   `updateSession` on `/api/*`; add the negative-lookahead so bearer-only requests
   get no session-cookie handling. Hard checklist item, not optional.
9. **Demo-mode rate limiting keys by IP**, not token — all demo callers share
   `lnx_demo.demo`, so token-keyed limiting throttles them as one bucket. Mirror
   `/api/demo/score`.
10. **SW message router checks `sender.id === chrome.runtime.id`** (+ a tab check)
    so only the extension's own popup/SW can trigger capture.
11. **Add an automated network-egress test** (stub `chrome.scripting` + `fetch`)
    asserting the service worker's only fetch host is the app origin — not just a
    manual DevTools check.
12. **`verifyBearer`: validate token shape** (has a dot, bounded length, base62
    id) **before** the DB lookup, and **never cache** a verification result (a
    revoked token must fail on the very next request).
13. **`sidePanel` permission gap** — if a side panel ships, add the `sidePanel`
    permission and re-justify; otherwise drop side-panel language and keep the
    popup only.
14. **Firefox:** `moz-extension://<id>` for the CORS allowlist is known only
    **after AMO signing** (pre-launch blocker); Chrome/Firefox `activeTab` /
    `host_permissions` grant semantics differ.

## 1. Mission & one-line scope

**Mission:** ship a Manifest V3 browser extension (Chrome/Edge primary, Firefox delta) that is a **manual capture + draft companion** for The Leads Nest.

**One-line scope:** on an explicit user click, capture **only the single public post currently on screen** (title, body, url, public author handle), send it to The Leads Nest app to score, render the score + breakdown, and let the user **copy a disclosed reply draft they post themselves** — nothing more.

**Out of scope, permanently:** background crawling, auto-posting/auto-DM, private/paywalled/DM access, browser automation, host-page mutation beyond the extension's own isolated UI. These are not "later" features; they are banned (Section 2).

## 2. SAFETY CONTRACT (embed verbatim in every extension phase — front and center)

> ### EXTENSION SAFETY CONTRACT — non-negotiable
>
> The Leads Nest browser extension is a **manual capture + draft companion**. It is **read-only to the host platform, write-only to the clipboard, single-post, and user-gesture-gated**. It extends the app's existing platform-safety law (`CLAUDE.md §2`, `docs/PLATFORM_SAFETY.md`, `docs/COMPLIANCE.md`). If a task drifts toward anything BANNED below, **STOP and flag it** — do not implement.
>
> **ALLOWED (and only this):**
> 1. On an **explicit user click** (toolbar action or single context-menu item), capture the **ONE** public post the user is **currently viewing** in the **active tab**.
> 2. Extract **only**: `platform`, `postUrl`, `postTitle`, `postBody` (excerpt, ≤5000 chars), `authorHandle` (public), `capturedAt`. Read-only DOM access.
> 3. Send that **single** payload over **HTTPS to The Leads Nest app origin only** for scoring (the existing Zod-validated, rate-limited, mock-on-demo `/api/demo/score` / `source_type:"manual"` path — never a paid provider on the public path, never persists on the public path).
> 4. Render the score + breakdown + a **disclosed** reply draft in the extension's **own popup/side panel**.
> 5. Let the user **COPY** the disclosed draft to the clipboard to post **themselves**. The draft passes `enforceReplySafety()` and always carries an affiliation disclosure + "you post this yourself" note.
>
> **BANNED (build will be rejected):**
> - Auto-post / auto-comment / auto-answer / auto-DM / any message send.
> - Programmatic clicks, focus, key/mouse/input events, `form.submit()`, or writing to host-page fields. The extension never mutates the host DOM beyond its own isolated UI.
> - Background crawling: opening tabs/iframes, `fetch`/`XHR`/`WebSocket` to any host platform, link-following, pagination, infinite-scroll, feed harvesting.
> - Capturing more than one post; bulk select; "scan this page"; local accumulation; CSV/JSON export.
> - Reading DMs, inboxes, notifications, private groups, paywalled or login-gated content, or any non-public surface.
> - Collecting data about other users (followers, member lists, profiles, emails).
> - Running without a user gesture (timers, observers, on-load capture).
> - Persistent/broad content scripts, `<all_urls>` host permissions, `debugger`/CDP, browser-automation libraries, remote code/`eval`, ToS/CAPTCHA evasion, identity rotation.
> - Reading/storing/transmitting host-platform cookies, tokens, or session credentials.
>
> **Technical controls (must all be present):** Manifest V3; permissions = `activeTab`, `scripting`, `clipboardWrite`, `storage` (+ optional `contextMenus`) **only**; **no** `content_scripts` auto-injection and **no** broad `host_permissions`; extraction injected via `chrome.scripting.executeScript` into the active tab **only at click time**; extractor returns **exactly one** record on the field allowlist and is read-only (no mutating DOM API); extension UI renders in a Shadow-DOM root (host-CSS isolation both ways); CSP `script-src 'self'` / `connect-src` limited to the app origin; capture **disabled** on private/DM/feed/non-permalink surfaces (public-permalink allowlist per platform); fail-safe = on any ambiguity, capture is disabled and nothing is sent.
>
> **CI gates (a violation fails the build):** static scan bans `.click()/.submit()/dispatchEvent/.focus()` on host nodes, `KeyboardEvent/MouseEvent/InputEvent`, `.value/.innerHTML/.textContent` assignment to host nodes, `execCommand`, `document.forms`, `eval`/remote code; grep bans `fetch`/`XHR`/`connect-src` to any host-platform domain and bans pagination/link-following helpers; dependency allowlist bans automation frameworks; manifest lint bans `debugger`, broad `host_permissions`, `<all_urls>`, and auto-injected content scripts.
>
> **Per-platform:** LinkedIn and X **automation is BANNED** — manual-view-only, single public post, read-only, no auto-actions, no crawling, capture disabled on feeds/DMs; honor their ToS. Reddit — single permalink, surface "subreddit self-promotion rules vary"; no private subreddits/modmail. HN/forums — single public permalink, respect `robots.txt`/ToS and community self-promotion rules.
>
> **Disclosures:** every draft discloses affiliation and says the user posts it; the extension shows what will be sent before sending; a published privacy policy with a single-purpose statement and minimal-permission justification ships with the Chrome Web Store listing.
>
> **Audit:** run `platform-safety-audit` + `security-compliance` on every extension change. Prove: gesture-gated, single-post, read-only-to-platform, clipboard-only, no host networking, no private surfaces, minimal data, disclosure present.

## 3. Architecture & file tree

Add a **sibling `extension/` workspace** in the repo so the Vite/CRX toolchain never collides with the Next.js build. The Next app (`:3000`) and the extension stay independently buildable. Reuse the repo's path-alias discipline and Crest tokens; the extension validates the app's response with Zod exactly as the server does (`CLAUDE.md §5`).

**Build tooling:** Vite + `@crxjs/vite-plugin` (understands MV3, gives HMR for popup + auto-reload for the service worker/content scripts, reads a typed `manifest.config.ts`, emits a clean `dist/` for load-unpacked). Tailwind via PostCSS, scoped to `./src`, reusing the Crest tokens. `esbuild` is a lighter alternative but you'd hand-roll the manifest emit + reload socket — `@crxjs` is the better fit.

```
extension/
```

**Component responsibilities:**
- **Service worker** (`background/service-worker.ts`) — the *only* component that talks to the network. Stateless message router; receives `CAPTURE_AND_SCORE` / `DRAFT_REPLY`; runs `chrome.scripting.executeScript` (gated by `activeTab`), receives the `CapturedPost`, `fetch`es the app, Zod-validates, returns to popup. Holds **no module-scope mutable state** (MV3 SW is ephemeral — see Section 3.1). Never touches the DOM.
- **Content script** (`content/capture.ts`) — NOT declared in `content_scripts`; a programmatic injection target only. Picks an adapter by hostname, extracts the single visible post, returns it as the `executeScript` result. Pure read; refuses (returns `{ blocked, reason }`) on the denylist.
- **Popup** (`popup/`) — the entire user-facing surface; where Crest design lives (Shadow DOM). Capture button → captured-post confirmation → score breakdown → disclosed draft → copy.
- **Options** (`options/`) — set app API origin (default from build-time env) + paste the extension API token. Stored in `chrome.storage.sync`/`local`.

### 3.1 MV3 gotchas the implementation MUST respect

- **Ephemeral service worker** (killed after ~30s idle): no module-scope mutable state you rely on later; read `chrome.storage` on each message; keep handlers self-contained. (The capture flow is already request-scoped.)
- **No remote code** (hard MV3 rule): all JS ships in the package; no `eval`/`new Function`/script-from-URL. CSP `script-src 'self'` enforces it. The mock scoring runs server-side, so there is nothing to fetch-and-execute.
- **Extension-page CSP**: popup/options are `script-src 'self'`; no inline `<script>`, no CDN. Tailwind ships static CSS. Verify the `dist` build loads the popup before shipping (Vite dev injects an HMR client; `@crxjs` handles the dev CSP carve-out).
- **`activeTab` lifetime**: grant lasts for that tab until navigation/close; run capture within the same user-gesture turn — don't defer injection behind an `await` chain that loses gesture context.
- **`executeScript` returns structured-clonable data only**: return a plain `CapturedPost`, never DOM nodes.
- **Clipboard**: `navigator.clipboard.writeText` needs a focused document + gesture — fine in the popup button handler; never attempt from the SW (no DOM).

## 4. `manifest.json` (MV3) — minimal permissions, each justified

Authored as a typed `extension/manifest.config.ts` (`defineManifest` from `@crxjs/vite-plugin`); the JSON below is the emitted shape.

```jsonc
{
  "manifest_version": 3,
  "name": "The Leads Nest — Capture & Draft",
  "version": "0.1.0",
  "description": "Manually capture the public post on screen, score buyer intent, and copy a disclosed reply you post yourself.",
  "minimum_chrome_version": "116",

  "permissions": [
    "activeTab",      // read the CURRENT tab's DOM, only after a user gesture, only on click. Replaces broad host access.
    "scripting",      // chrome.scripting.executeScript to inject capture.ts on demand (no static content_scripts).
    "storage",        // persist the app API origin + token + UI prefs in chrome.storage. No post content stored.
    "clipboardWrite"  // copy the disclosed reply draft for the user to paste. The ONLY "write" the extension performs.
    // "contextMenus" // OPTIONAL second capture entry point ("Score this post"). Add only if shipping the menu item.
  ],

  // ONLY the app's own API origin. NOT the social platforms — those are reached via activeTab DOM read, never network.
  "host_permissions": [
    "http://localhost:3000/*",
    "https://app.theleadsnest.com/*"
  ],

  // For a USER-CONFIGURABLE self-hosted origin, request it at runtime ONLY for the origin the user types in Options.
  "optional_host_permissions": [],

  "background": { "service_worker": "src/background/service-worker.ts", "type": "module" },

  "action": {
    "default_title": "Capture this post for The Leads Nest",
    "default_popup": "src/popup/index.html",
    "default_icon": { "16": "icons/icon-16.png", "32": "icons/icon-32.png" }
  },

  "options_page": "src/options/index.html",
  "icons": { "16": "icons/icon-16.png", "48": "icons/icon-48.png", "128": "icons/icon-128.png" },

  // MV3 default extension-page CSP. No remote code, no inline script, no eval.
  "content_security_policy": { "extension_pages": "script-src 'self'; object-src 'self'" }
}
```

**Why each permission, and what is deliberately absent:**
- `activeTab` — temporary read access to the tab **only at the moment of the action click**, only that tab. With no static `content_scripts` and no broad host match, the extension has *zero* ability to see any page until you click. This is exactly why we **reject `<all_urls>`** (it would let a content script run on every page automatically — the definition of background crawling, banned by `CLAUDE.md §2`).
- `scripting` — to programmatically inject `capture.ts` under `activeTab`. Without it you'd need static `content_scripts` (auto-injection) — the thing we avoid.
- `storage` — only app origin + token + UI prefs. **No captured post content is ever persisted.**
- `clipboardWrite` — the single write capability, used solely to copy a disclosed draft. (In a popup with a gesture, `navigator.clipboard.writeText` often works without it, but declaring it makes copy robust across browsers.)
- `host_permissions` = **app origin only.** The extension never network-calls the platforms; it reads them via on-screen DOM under `activeTab`. So the platforms need **no** host permission — a strong signal to reviewers and users that this is not a scraper.

**Explicitly NOT requested (each maps to a `CLAUDE.md §2` prohibition):** `<all_urls>`/broad host matches (background crawling), `tabs` (enumerate/track other tabs), `webRequest`/`declarativeNetRequest` (traffic interception → automation/evasion), `cookies` (touching platform sessions), `history`, `downloads`, `alarms` (scheduled background scans), `notifications`, `debugger` (CDP/automation).

## 5. Backend: `/api/extension/*` routes, auth, contracts

The routes are a thin, token-authenticated front door to the **existing pipeline** — `scoreManualPost()` and `generateReplyForLead()` in `src/lib/scan.ts`, which already run keyword filtering, AI scoring, usage-limit checks, persistence, and `enforceReplySafety()`. The routes add **no new scoring/safety logic** and therefore inherit every platform-safety guarantee. Both are `POST` (+ `OPTIONS`), `export const dynamic = "force-dynamic"`, `runtime = "nodejs"` (token hashing needs `node:crypto`).

> **Note on the demo target:** the existing public `POST /api/demo/score` (`src/app/api/demo/score/route.ts`) is mock-only, per-IP rate-limited, and persists nothing — its request body uses `{ product, keywords, competitors, negativeKeywords, postTitle, postBody (1..5000) }` and its response is `{ overall, scores{relevance,intent,urgency,fit}, lead_stage, reason, pain_points, buying_signals, suggested_angle, risk_flags }`. The extension's **signed-out / demo path** targets this endpoint unchanged. The **signed-in path** targets the new `/api/extension/*` routes below, which run the org's real ICP through the authenticated pipeline.

### 5.1 Auth — recommended: per-user revocable Extension API token (NOT session cookie)

Authenticate with a **per-user EXTENSION API TOKEN minted in `/app/settings`**, sent as `Authorization: Bearer lnx_<id>.<secret>` (mirrors the `CRON_SECRET` bearer precedent in `src/app/api/cron/digest/route.ts`). **Reject session-cookie auth:** cross-site cookies are dropped by `SameSite=Lax` + third-party-cookie deprecation, a background SW has no reliable cookie jar, cookie+cross-origin-POST is the classic CSRF surface, cookies rotate on Supabase's refresh cycle (the extension can't refresh them), and there is **no real session in demo mode** (auth is a stubbed `DEMO_USER`). The bearer token is origin-independent, CSRF-free (not auto-attached), independently revocable, scoped to exactly capture+draft, hashable at rest, and works in demo mode via a seeded token.

**Token format/minting/storage:** `lnx_<tokenId>.<secret>` — `tokenId` is a short public lookup id (12-char base62), `secret` is 32 bytes CSPRNG (`crypto.randomBytes(32)`, base64url). **Shown once** at mint time (like a GitHub PAT). Store only `token_id` (plaintext, O(1) lookup) + `token_hash = sha256(secret)` — the secret has full entropy so plain SHA-256 is sufficient and constant-time-comparable; never store the secret or full token. **Verify:** split on `.` → look up by `token_id` → `crypto.timingSafeEqual(sha256(presentedSecret), stored_hash)` → check `revoked_at IS NULL` + optional `expires_at` → bump `last_used_at` (fire-and-forget) → return `{ userId, organizationId, tokenId }`. Routes look up tokens with the **service-role/admin store** (no session in the request) — **app-level org-scoping is the first line of defense; RLS is the second.** The org id comes *out of* the verified token, never from the request body.

**Migration `supabase/migrations/0006_extension_tokens.sql`** (follow the `0002_rls.sql` pattern; use the next free migration number in the repo):

```sql
create table public.extension_api_tokens (
  id              uuid primary key default gen_random_uuid(),
  organization_id uuid not null references public.organizations(id) on delete cascade,
  user_id         uuid not null references auth.users(id) on delete cascade,
  token_id        text not null unique,          -- public lookup id (part before the dot)
  token_hash      text not null,                 -- sha256(secret)
  name            text,                          -- user label, e.g. "Chrome at home"
  last_used_at    timestamptz,
  expires_at      timestamptz,
  revoked_at      timestamptz,
  created_at      timestamptz not null default now()
);
create index extension_api_tokens_org_idx on public.extension_api_tokens(organization_id);
alter table public.extension_api_tokens enable row level security;
create policy ext_tokens_select_member on public.extension_api_tokens
  for select using (public.is_org_member(organization_id));
create policy ext_tokens_insert_member on public.extension_api_tokens
  for insert with check (public.is_org_member(organization_id) and user_id = auth.uid());
create policy ext_tokens_update_member on public.extension_api_tokens
  for update using (public.is_org_member(organization_id));
```

**Five new `DataStore` methods (both `MemoryStore` + `SupabaseStore`, kept at parity):** `createExtensionToken(orgId, userId, {name, token_id, token_hash, expires_at?})`, `findExtensionTokenById(token_id)`, `touchExtensionToken(id)`, `revokeExtensionToken(orgId, id)`, `listExtensionTokens(orgId)`.

### 5.2 `POST /api/extension/capture`

Maps onto `scoreManualPost(store, orgId, plan, project, manualSourceId, input)`. Sequence: `OPTIONS` short-circuit (CORS) → verify bearer → per-token rate limit → Zod-validate → `store = await getStore()` + org `subscription.plan` → resolve project (`body.project_id` validated via `assertProjectBelongsToOrg` (IDOR guard), else the org's first project; `404 project_not_found` if none) → find the org's `manual` source (`listSources(...).find(s => s.source_type === "manual")`, as `addManualPostAction` does) → `scoreManualPost(...)` (`limitReached` → `402`, `error` → `502`) → `200` with `lead_id` + breakdown. The `manual_` external-id + `upsertRawPost` dedupe in `scoreManualPost` makes re-capture idempotent.

**Request (Zod, mirrors the demo route's caps):**
```jsonc
{
  "post": {
    "title":         "string|null",   // .max(500).optional
    "body":          "string",         // .min(1).max(5000) — required
    "url":           "string|null",    // .url().max(2000).nullish — the public permalink
    "author_handle": "string|null"     // .max(200).nullish — public handle only (maps to author_display)
  },
  "project_id": "uuid|null",
  "platform":   "reddit|hackernews|rss|other"
}
```

**Response `200`:**
```jsonc
{
  "api_version": "2024-06-01",
  "lead_id": "uuid",
  "overall": 0,
  "scores": { "relevance": 0, "intent": 0, "urgency": 0, "fit": 0 },
  "lead_stage": "string", "reason": "string",
  "pain_points": ["string"], "buying_signals": ["string"],
  "suggested_angle": "string", "risk_flags": ["string"],
  "usage": { "posts_scanned": { "used": 0, "limit": 0, "remaining": 0 } },
  "app_url": "https://app.theleadsnest.com/app/leads/<lead_id>"
}
```
Shape intentionally matches `/api/demo/score` + `lead_id`/`usage`/`app_url`, so the extension renders demo and signed-in results with the same code.

### 5.3 `POST /api/extension/draft`

Maps onto `generateReplyForLead(store, orgId, plan, leadId, toneOverride)`. Sequence: auth → rate limit → Zod-validate `{ lead_id, tone? }` → `assertLeadBelongsToOrg(lead_id, orgId)` (IDOR guard) → `generateReplyForLead(...)` (`limitReached` → `402`). The draft already passed `enforceReplySafety()` so `suggested_disclosure` is guaranteed non-empty.

```jsonc
// request
{ "lead_id": "uuid", "tone": "helpful|friendly|professional|concise|null" }
// response 200
{
  "api_version": "2024-06-01",
  "reply_id": "uuid", "reply_text": "string",
  "suggested_disclosure": "string",      // GUARANTEED non-empty by enforceReplySafety()
  "why_this_reply": "string",
  "safety_notes": ["Disclose your affiliation before mentioning your product.", "..."],
  "confidence": 0.0,
  "platform_reminder": "The Leads Nest does not post for you. Review the platform's rules before responding.",
  "usage": { "reply_drafts": { "used": 0, "limit": 0, "remaining": 0 } }
}
```
Two routes (not one) because `posts_scanned` and `reply_drafts` are separately metered in `checkUsage` — capture+score many posts cheaply, spend a draft credit only when the user actually intends to reply.

### 5.4 `GET /api/extension/health` (optional, recommended)

Token check + plan/quota probe the options page calls after a token is pasted: `{ ok, demo, plan, remaining: { posts_scanned, reply_drafts } }` or `401`. Lets Options show "Connected as <org> · 412 scans left" instead of failing on first capture.

### 5.5 Org-scoping, rate limit, demo mode, CORS

- **Org-scoping:** token verify yields `{ userId, orgId }`; every `project_id`/`lead_id` is checked with `assertProjectBelongsToOrg`/`assertLeadBelongsToOrg`. `AuthorizationError` → `401`/`404`, never `500`. Never trust a client-supplied org id.
- **Rate limit:** reuse `rateLimit()` (`src/lib/ratelimit.ts`), keyed by **token id** (stable per-user identity, immune to NAT collisions): `capture` `{ limit: 30, windowMs: 60_000 }`, `draft` `{ limit: 15, windowMs: 60_000 }`. On limit → `429` + `Retry-After: 60` + `code: "rate_limited"`. The monthly `checkUsage` plan quota is the **hard backstop** regardless of window. Low ceilings + per-call manual trigger + **no list/feed endpoint** enforce "manual capture, not a crawler" at the API layer.
- **Demo mode (`isDemoMode()`):** accept a fixed seeded token (`lnx_demo.demo`) resolving to `DEMO_USER_ID`/`DEMO_ORG_ID` (`src/lib/db/seed.ts`); `getStore()` returns `MemoryStore`; deterministic mock provider. The `/app/settings` mint UI shows the seeded demo token read-only. Zod + rate limiting still run, so the contract is identical offline. **The entire capture→score→draft loop runs with zero secrets.**
- **CORS for `/api/extension/*`:** answer `OPTIONS` with `Access-Control-Allow-Origin` echoed from an **allowlist of known extension IDs** (`chrome-extension://<id>`, `moz-extension://<id>`) — never `*`; `Allow-Methods: POST, OPTIONS`; `Allow-Headers: Authorization, Content-Type, X-LeadsNest-Ext-Version`; `Max-Age: 86400`; **no `Allow-Credentials`** (bearer tokens, not cookies). Set headers in a tiny `withExtensionCors()` wrapper, not middleware. **Exclude `/api/extension` from `src/middleware.ts`** session refresh (negative-lookahead in the matcher) — these routes have no cookie session.
- **Error envelope (all non-2xx):** `{ "api_version": "...", "error": { "code": "limit_reached", "message": "Human-readable, safe to display." } }`. Codes: `invalid_input`(400), `unauthorized`(401), `limit_reached`(402), `project_not_found`/`lead_not_found`(404), `method_not_allowed`(405), `payload_too_large`(413), `rate_limited`(429), `internal_error`(500), `scoring_failed`(502). Route all unexpected throws through `captureError(err, { scope: "extension-capture", orgId, tokenId })`; never leak internals.
- **Versioning:** stable URLs; `api_version: "2024-06-01"` in every response + `X-LeadsNest-API-Version` header. Extension sends `X-LeadsNest-Ext-Version`; server may return a soft `warning` or hard-fail `426 upgrade_required` below a min version. Breaking changes go to `/api/extension/v2/*`, never mutate v1.

**Backend files:** `src/app/api/extension/{capture,draft,health}/route.ts` (NEW); `src/lib/extension/auth.ts` (NEW — `mintToken`, `verifyBearer`, hashing, demo-token shortcut); `src/lib/extension/cors.ts` (NEW — `withExtensionCors` + ID allowlist); `supabase/migrations/0006_extension_tokens.sql` (NEW); `src/lib/db/{store,memory-store,supabase-store}.ts` (EDIT — 5 methods); `src/lib/actions.ts` (EDIT — `createExtensionTokenAction`/`revokeExtensionTokenAction`, guarded by `requireContext` + ownership); `src/app/app/settings/page.tsx` (EDIT — mint/list/revoke UI); `src/middleware.ts` (EDIT — exclude `/api/extension`).

## 6. UX flows + Crest styling + states

**Design substrate (reuse, do not re-skin):** render all UI inside a **Shadow DOM root** (`ExtRoot`) so host CSS can't bleed in and the extension can't restyle the host page (enforces C7/B14). Ship a scoped copy of the app's `globals.css` tokens + utilities (`.card`, `.btn-primary/secondary/ghost`, `.badge`, `.input`, `.label`, `--paper/--ink/--accent/--accent-tint/--on-accent`, `paper-sunk`, `line/line-2`, `surface`). Fonts bundled in-package (Newsreader display, IBM Plex Sans body, IBM Plex Mono UPPERCASE tracked labels) — **no font CDN call from a content script.** Crest invariants everywhere: Cloud-Dancer paper, ONE Verdigris accent, **1px borders not shadows**, 0px corners on cards / full-pill on chips+buttons, mono uppercase `tracking-caps` labels, **no emoji, no hype**. Tiers reuse `scoreTier`/`tierMeta`: high ≥70, medium 40–69, low <40.

**Two user-invoked surfaces only — nothing runs without a click.**

**SURFACE A — In-page button (content script, user-invoked).** A single injected pill anchored near the focused public post's own action row: 1px Verdigris border, paper fill, `BrandMark` glyph + mono caps label `SCORE THIS POST`, no shadow. Click is the only capture trigger; it reads ONLY `title`/`body`/`url`/`author_display` from the anchored node (no comments, no siblings, no network beyond the app), then opens Surface B beside it without reflowing the host post. If the node yields no usable text → button flips to `COULDN'T READ THIS POST` (mono caps, `text-low`) and never silently grabs a different post. Unsupported page → no button injected (discovery lives in the popup). *(Surface A is optional for v0.1 — the popup, Surface C, can host the entire flow; ship A in the polish phase.)*

**SURFACE B — Capture & Score panel (core flow).** A `360×auto` Crest `.card` (0px corners, 1px `line-2`, `surface` fill) in a Shadow root, also used as the popup body (same component):
- **B0 Header (persistent):** Wordmark left (18px), mono caps `CAPTURE · DRAFT` tag right.
- **B1 Loading (right after capture click):** the **CapturedPostCard shown first** — mono caps source chip row (`REDDIT · R/AGENCY · U/MAPLE · PERMALINK`) + Newsreader 2-line-clamp title — then `◢ SCORING THIS POST…` with a 1px indeterminate accent track and the reassurance line `We only read this one public post.` (Transparency: the user sees exactly what was/ will be sent.)
- **B2 Score result:** Newsreader 5xl `overall` (`text-accent` for high) + `ScoreBadge` pill; `ScoreBars` reused verbatim (35/30/20/15 weights, mono caps label·weight, 1px `paper-sunk` track, `accent-soft` fill, tabular value); `── WHY` reason; `STAGE` `StageBadge`; `SIGNALS` ghost chips; `⚠ RISK FLAGS` (only if present; `border-l-2 border-accent` `accent-tint`, SafetyNotice style — e.g. "Reddit self-promotion rules vary by subreddit"); actions `[ DRAFT A REPLY ]` (primary) `[ OPEN IN APP ↗ ]` (secondary, deep-links `/app/leads/<id>` signed-in or `/demo` in demo); persistent `SafetyNotice` footer "We don't post anything. You review and post replies yourself." **Low-score path:** same layout, tier pill `LOW`, calm ink-3 line "Low buyer intent — probably skip this one.", `DRAFT A REPLY` de-emphasized to secondary (never hidden — user stays in control).
- **B3 Draft reply (copy-only):** the **copy block is structurally inseparable from the disclosure** — `reply_text` and `suggested_disclosure` render inside one bordered card labeled "you'll post this yourself", with `── DISCLOSURE (keep this)` pinned inside. `TONE` segmented pills (`helpful|concise|warm`, ghost→accent active) regenerate via the draft route's `tone` override; `── WHY THIS REPLY`; `SAFETY NOTES` mono "·" bullets (`enforceReplySafety` guarantees the disclosure note); `[ COPY REPLY + DISCLOSURE ]` (primary) **always concatenates `reply_text` + blank line + `suggested_disclosure`** — there is **no UI path to copy the reply without the disclosure**; `[ REGENERATE ]` (ghost). On copy success the label swaps to `COPIED — NOW PASTE IT YOURSELF` (mono caps accent, no emoji — a 1px accent underline is the confirmation). Demo drafts carry an `enforceReplySafety`-style disclosure and a `DEMO DRAFT` tag.

**SURFACE C — Toolbar popup (account, status, fallbacks), 360 wide.** When a supported post is on the active tab it renders Surface B inline; otherwise:
- **C1 Signed-in idle:** Wordmark + account email; `PUBLIC POST DETECTED ON THIS PAGE` + Newsreader preview; `[ CAPTURE + SCORE THIS POST ]` (primary, full); `PROJECT [ Agency Tools ▾ ]` (the ICP that scores it); SafetyNotice "We read only the post you choose, when you click. Nothing auto-runs."
- **C2 Not signed in:** "Sign in to score with YOUR product's ICP and save leads." `[ SIGN IN ↗ ]` (opens app `/login` in a tab — auth happens in the app, never in-ext) + `[ TRY DEMO SCORING ]` (runs the full capture→`/api/demo/score`→Surface B flow with the seeded demo product, like `DemoSearch` defaults — **fully clickable with zero secrets**).
- **C3 Unsupported page:** `NO PUBLIC POST HERE` EmptyState (BrandMark icon) "Open a public thread (Reddit, Hacker News, a public forum) and try again." + `[ PASTE A POST ↗ ]` (opens app manual-paste/demo); SafetyNotice "We never read private or logged-in content."
- **C4 Capture failed:** `COULDN'T READ THE POST` + a transparency card listing the only 4 fields `TITLE · BODY · URL · PUBLIC HANDLE`; `[ TRY AGAIN ]` + `[ PASTE INSTEAD ↗ ]`.
- **C5 Rate-limited (maps to 429):** `TOO MANY SCORES RIGHT NOW / Give it a minute and try again.` `[ TRY AGAIN ]` disabled honoring `Retry-After`.
- **C6 Demo-mode banner (global):** persistent mono caps strip `DEMO MODE · SAMPLE PRODUCT · NOTHING SAVED` in `accent-tint` with `border-accent` (SafetyNotice styling).

**State machine (one flow, all surfaces):** `detect page` → (unsupported→C3 | signed-out→C2 | signed-in→C1/B inline) → user clicks CAPTURE (only trigger) → `B1 LOADING` (capture fail→C4) → `POST score` (429→C5) → `B2 SCORE` (OPEN IN APP↗→app) → `DRAFT A REPLY` → `B3 DRAFT` (TONE/REGENERATE) → `COPY REPLY + DISCLOSURE` → `COPIED — PASTE IT YOURSELF` (terminal; user posts manually). **No node posts, sends, DMs, or fetches a second post.**

**Component reuse:** ship the same source into the extension bundle — `ScoreBadge`/`StageBadge`/`ScoreBars` (`src/components/score-badge.tsx`), `ActionButton` (`src/components/crest/action-button.tsx`), `BrandMark`/`Wordmark` (`src/components/crest/brand-mark.tsx`), `SafetyNotice`/`EmptyState`/`Badge`/`Card`/`SectionTitle` (`src/components/ui.tsx`), `scoreTier`/`tierMeta`/`computeOverallScore`/`HIGH_INTENT_THRESHOLD` (`src/lib/scoring/score.ts`), `enforceReplySafety`/`replyDraftSchema` (`src/lib/ai/safety.ts`, `src/lib/ai/schemas/reply-draft.ts`). New extension-only components: `ExtRoot`, `InPageScoreButton`, `CapturedPostCard`, `CapturePanel`, `ReplyDraftPanel`, `ToneToggle`, `PopupShell`, `DemoModeBanner`, and the pure reader `capturePost()`.

## 10. Definition of done + gates that must pass

**Definition of done:**
1. `extension/dist` loads unpacked in Chrome/Edge; the full **capture → score → draft → copy** loop works in **demo mode with zero secrets**, and signed-in against `/api/extension/*` with a minted token.
2. The manifest requests **only** `activeTab`, `scripting`, `storage`, `clipboardWrite` (+ optional `contextMenus`), `host_permissions` = app origin only, **no `<all_urls>`**, **no auto-injected content scripts**.
3. Capture is reachable **only** via a user gesture, reads **exactly one** post and **only** the 4 public fields, is **disabled** on private/DM/feed surfaces, and the SW makes exactly one request per capture — to the app origin only.
4. The copy action is **structurally incapable** of omitting the affiliation disclosure; every draft carries it via `enforceReplySafety()` + the "you post this yourself" note; a persistent `SafetyNotice` shows on score and draft views.
5. Backend routes are token-authenticated, org-scoped (IDOR-guarded), per-token rate-limited, Zod-validated, demo-mode-safe, with the CORS extension-ID allowlist and middleware exclusion in place; the new table has RLS and both stores at parity; the mint/revoke UI works (show-once token).
6. Privacy policy + single-purpose statement + per-permission justification authored; Firefox delta builds.

**Gates that must pass (all green, no exceptions):**
- `release-gate` — verify:env · typecheck · lint · test:coverage · build · e2e — on the Next app **and** the `extension/` workspace.
- `platform-safety-audit` — proves gesture-gated, single-post, read-only-to-platform, clipboard-only, no host networking, no private surfaces, minimal data, disclosure present; the Section 2/9 CI static-scan + manifest-lint + dependency-allowlist gates pass with zero violations.
- `security-compliance` — token hashing (timing-safe, hashed at rest), CORS allowlist tight (never `*`, no credentials), IDOR guards on every `project_id`/`lead_id`, no secret leakage in error messages, no remote code in the bundle.
- `crest-design` — UI matches The Crest (Cloud-Dancer paper, one Verdigris accent, Newsreader+Plex, 0px/pill corners, 1px borders, mono uppercase labels, no emoji/hype) and is fully Shadow-DOM isolated.
- **Human approval** before any merge to `main`, any Supabase migration apply, or any store submission.

**Known limitations to flag for the human:** (1) the in-memory `rateLimit()` is per-instance/best-effort on multi-instance Vercel — the monthly `checkUsage` quota is the hard backstop; slot Upstash/Redis behind the same API for strict global limits. (2) Token verify uses the service-role/admin store (no session in the request), so app-level org-scoping is the primary guard and RLS is defense-in-depth — routes must never trust a client-supplied org/project id. (3) The CORS extension-ID allowlist needs the real published store IDs before launch. (4) Reddit/HN/generic adapter selectors need real-DOM validation and will drift as platforms change markup. (5) Surface A (in-page button) is optional for v0.1 — the popup hosts the full flow; ship A in E6. (6) No `extension/` directory exists in the repo yet — this is the pre-build governance + execution spec for it.
