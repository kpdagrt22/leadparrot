## 1. Testing philosophy & non-negotiable rules

These rules are invariant. A PR that violates any of them is rejected regardless of green CI.

1. **Deterministic only.** No real clocks, no real randomness, no network. Inject `now` (see `tests/unit/ratelimit.test.ts`), seed any randomness, and stub `fetch`/adapters. A test that can flake is a bug.
2. **Mock/demo only — NEVER real third-party keys.** Every test runs with `LEADPARROT_DEMO=1` semantics or injected mock clients. Tests MUST pass on a machine with **zero** Supabase/AI/Stripe/Resend/Reddit secrets. If a test needs a key to pass, it is wrong. The *only* exception is the optional RLS test (P6), which is gated behind `SUPABASE_TEST_DB` and **skips** when unset — it never requires a cloud key.
3. **Demo-mode E2E.** Playwright drives the app under `LEADPARROT_DEMO=1` (in-memory `MemoryStore`, mock AI, seeded data, `enter demo workspace`). E2E never touches a real backend.
4. **Mock-first AI, never fabricate a high-intent lead.** Any AI path under test must degrade to the mock provider and produce schema-valid, in-range output. Malformed provider output must fall back without ever inventing a relevant/high-intent lead (see `ai-service-fallback.test.ts`, `lead-scoring-schema.test.ts`).
5. **Regression-test every bug.** Every fixed defect ships with a failing-first test that pins the corrected behavior. No bug is "done" without it.
6. **Safety-by-design assertions are mandatory, not optional.** Owner-only notifications, copy-only replies (no auto-post), no scraping, server-derived org IDs, signature-verified webhooks — each must have an explicit assertion. Removing a safety assertion requires sign-off.
7. **Report REAL output. Never claim green you did not see.** Paste the actual terminal output of `npm run test:coverage` and `npm run test:e2e` (the coverage summary table and the Playwright pass/fail lines). "Tests pass" without pasted output is not acceptable. If you could not run it, say so explicitly.
8. **Mirror the existing conventions.** New unit tests live in `tests/unit/**/*.test.ts` (Vitest, `globals: true`, `node` env, `@` → `./src`). New e2e specs live in `tests/e2e/*.spec.ts`. Route-handler tests import the `POST`/`GET` export directly and inject mock clients — they do **not** boot a server.
